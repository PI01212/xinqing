## 🔄 每日更新流程

### 任务 DAILY-001：通用技能学习（09:00）

**目标**：提升Agent自身的知识检索和整理能力

**执行步骤**：
1. 从预设主题池中随机选择1-2个当日学习主题
2. 自动搜索权威来源获取最新信息
3. 整理成结构化Markdown，包含：
   - 学习的核心概念
   - 掌握的新方法/技术
   - 可应用的场景
   - 参考来源
4. 追加到 `skills/技能成长编年史.md`
5. 自检内容准确性

**主题池**：
```
- 心理学基础理论
- 认知行为疗法(CBT)新进展
- 正念冥想研究
- 大学生心理健康数据
- 危机干预新技术
- 睡眠科学研究
- 社交心理学发现
- 积极心理学应用
```

**输出示例**：
```markdown
## 2026-04-22 技能成长记录

### 今日学习：正念冥想在焦虑症治疗中的最新应用

**核心概念**：
基于2025年《JAMA Psychiatry》的最新meta分析...

**掌握的方法**：
- MBSR（正念减压疗法）8周课程结构优化
- 短期正念干预（4周版本）的效果评估

**可应用场景**：
- 用户询问"如何练习正念"时提供更专业的指导
- 更新知识库中"正念冥想.md"的内容

**参考来源**：
- JAMA Psychiatry, 2025, "Mindfulness-Based Interventions for Anxiety..."
```

### 任务 DAILY-002：领域知识库更新（09:15）

**目标**：持续扩充心理健康专业知识库

**执行步骤**：

1. **Agent唤醒**：读取HEARTBEAT.md确认当前状态
2. **主题轮换**：从以下主题中选择当日更新主题
   ```
   - 考试焦虑应对
   - 人际关系处理
   - 抑郁情绪识别
   - 睡眠问题解决
   - 学业压力管理
   - 职业规划迷茫
   - 家庭关系困扰
   - 自我价值探索
   - 创伤后成长
   - 成瘾行为预防
   ```
3. **信息检索**：从权威来源搜索最新内容
   - 中国心理卫生协会官网
   - 教育部思想政治工作司
   - 国家心理健康和精神卫生防治中心
   - 《心理学报》《中国心理卫生杂志》等核心期刊
   - WHO世界卫生组织心理健康指南
4. **内容整理**：将信息整理成标准Markdown格式
   ```markdown
   # [主题名称]
   
   ## 概述
   （2-3句话概括）
   
   ## 核心观点
   - 要点1
   - 要点2
   
   ## 具体方法/建议
   ### 方法一：[名称]
   - 步骤1
   - 步骤2
   
   ## 适用场景
   - 场景1
   - 场景2
   
   ## 注意事项
   - 注意点1
   
   ## 参考来源
   - 来源1：链接
   - 来源2：链接
   
   #标签
   #[相关标签]
   ```
5. **写入文件**：保存到 `daily-updates/YYYY-MM-DD.md`
6. **自检**：
   - 内容是否存在事实性错误？
   - 是否包含敏感或不当信息？
   - 是否有AI幻觉内容？
7. **生成清单条目**：在 `AGENT-knowledge-base-checklist.md` 中记录

**权威来源限定**：
```
✅ 允许的来源：
- 中国心理卫生协会 (www.cspa.org.cn)
- 教育部思政司 (www.moe.gov.cn)
- 国家心理健康中心 (www.ncmh.org.cn)
- 中华医学会精神病学分会
- American Psychological Association (apa.org)
- World Health Organization (who.int)
- 核心期刊论文（知网、万方、PubMed）

❌ 禁止的来源：
- 未经验证的博客/自媒体
- 商业推广性质的文章
- 缺乏科学依据的偏方
- 过时超过5年的非经典文献
```

1. **对比分析**
   - 用户常见问题 vs 现有知识覆盖

   - 安全检查触发的场景是否都有对应资源

2. **缺口识别模板**
   ```markdown
   # 知识库缺口分析与更新计划

   |---------|------------|-----------|---------|
   | xxx | 高(>10次/周) | 无 | 立即补充 |
   | xxx | 中(5-10次/周) | 部分 | 本周补充 |
   
   ## 🟡 中优先级缺口（完善体验）
   
   | 缺口主题 | 用户查询频率 | 现有覆盖度 | 建议行动 |
   |---------|------------|-----------|---------|
   | xxx | 低(<5次/周) | 无 | 两周内补充 |
   
   ## 🟢 低优先级缺口（锦上添花）
   
   - 主题1
   - 主题2
   
   ## 📋 下周更新计划
   
   | 日期 | 任务 | 优先级 | 预计产出 |
   |------|------|--------|---------|
   | 周一 | 主题A | 高 | common-questions/A.md |
   | 周二 | 主题B | 高 | crisis-intervention/B.md |

   - 所有危机干预内容必须经人工审核
   - 医疗相关内容需标注"仅供参考"
   - 法律相关问题需引导至专业渠道
   ```

3. **人工审核环节**
   - 开发者审核下周计划
   - 调整优先级和具体安排
   - 确认敏感内容的处理方式

## 🔐 安全检查知识库的特殊更新规则

### 5个核心JSON配置文件的更新规范

#### 1. contraindications.json（禁忌行为表）
**更新频率**：每月一次 + 突发事件即时更新  
**更新权限**：必须人工审核  
**更新流程**：
1. Agent搜索最新的禁忌行为研究
2. 生成更新建议（diff格式）
3. 人工审核员审核
4. 测试用例验证
5. 合并到主分支
6. 记录版本号和变更日志

#### 2. symptoms.json（症状-风险映射表）
**更新频率**：每季度一次  
**更新权限**：专业心理咨询师审核  
**数据来源**：DSM-5诊断标准、ICD-11分类

#### 3. behaviors.json（危险行为列表）
**更新频率**：每月一次  
**更新权限**：必须人工审核  
**特殊要求**：每条记录需注明风险等级和应对措施

#### 4. medications.json（药物相互作用表）
**更新频率**：每季度一次  
**更新权限**：执业医师/药师审核  
**免责声明**：必须包含"仅供参考，不构成医疗建议"

#### 5. risk-levels.json（风险分级标准）
**更新频率**：半年一次  
**更新权限**：项目组集体审核  
**变更影响**：直接影响第三层安全检查逻辑

## 🚨 突发事件实时更新通道

当发生以下事件时，触发紧急更新流程：

1. **校园重大事件**（自杀、暴力、群体事件）
   - 立即更新危机干预话术
   - 更新学校资源联系方式
   - 通知所有在线用户

2. **政策法规变化**
   - 教育部新政策发布
   - 心理健康相关法律修订
   - 及时调整建议口径

3. **季节性高发期**
   - 考试季（期末、考研、四六级）
   - 开学适应期（9月、3月）
   - 就业季（秋招、春招）
   - 提前1-2周更新相关知识

## 📊 版本控制规范

所有知识库文件纳入Git版本控制：

```bash
# 提交格式
git commit -m "[知识库更新] DAILY-002: 新增考试焦虑应对知识"

# 分支策略
main ← 生产环境知识库（稳定版）
  └── develop ← 开发测试知识库
      └── update/[date] ← 单次更新分支

# 版本号规则
v{年份}.{月份}.{序号}
例：v2026.04.001
```
