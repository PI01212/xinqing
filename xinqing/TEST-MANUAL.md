## 🧪 测试一：长期记忆功能

### 测试目标
验证系统能够：
1. ✅ 识别并保存用户基本信息（姓名、年龄、年级、专业等）
2. ✅ 记录用户情绪变化
3. ✅ 记录用户压力来源
4. ✅ 生成核心记忆
5. ✅ 在后续对话中调用记忆

### 测试用例

#### TC-001：用户自我介绍记忆

**目的**：验证系统能否从对话中提取并保存用户基本信息

**测试步骤**：
1. 连接WebSocket
2. 发送以下消息（每次一条）：

```

我今年20岁
我是大二学生
我的专业是计算机科学
我在清华大学读书
```

**预期结果**：
```
[记忆] 👤 记录用户姓名: 李明
[记忆] 🎂 记录用户年龄: 20岁
[记忆] 📚 记录用户年级: 大二
[记忆] 🎓 记录用户专业: 计算机科学
[记忆] 🏫 记录用户学校: 清华大学
```

**验证方法**：
- 查看 `data/users/{userId}/profile.json` 文件
- 确认所有信息都被正确保存

## 🔍 测试结果分析

### 如果所有测试都通过 ✅
恭喜！你的长期记忆和知识库功能已完全实现且运行正常。

### 常见问题排查

#### ❌ 知识库检索返回空结果

**可能原因**：
1. 知识库文件路径不正确
2. 中文分词问题
3. 知识库未正确加载

**排查方法**：
```bash
# 检查知识库文件是否存在
ls d:\laji\ai_project\xinqing\knowledge\

# 检查服务器启动时的知识库加载日志
# 应该看到类似输出：
# [知识库] 📚 知识库加载完成，共 14 个条目
```

#### ❌ 记忆功能不工作

**可能原因**：
1. `data` 目录没有写入权限
2. 用户ID每次都不同（WebSocket重连后）

**排查方法**：
```bash
# 检查data目录
ls d:\laji\ai_project\xinqing\data\users\

# 应该看到以 "test-user-" 或 "xinqing-" 开头的目录
```

**重要**：WebSocket连接使用的是随机生成的deviceId，如果你每次连接都断开重连，系统会创建新的用户数据。

#### ❌ AI回复没有引用知识库内容

**排查方法**：
1. 检查服务器日志中的 `[上下文]` 输出
2. 确认知识库检索有结果（`✓ 知识库检索成功`）
3. 检查 `context.knowledgeContext` 是否被构建
