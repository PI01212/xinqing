第一部分：选题过程
1.1 备选题目
在日常学习与生活中，通过对周围同学以及自身痛点的细致观察，我发现大学生群体在心理健康、学习规划与健康饮食三个方面面临着显著的挑战。基于这些观察，我提出了三个备选的智能应用项目方向。

第一个备选项目是“心晴——大学生心理健康陪伴助手”。当前大学生面临学业、就业以及人际交往等多重压力，心理健康问题日益突出。然而，传统的心理咨询服务往往门槛较高且缺乏即时性。该项目旨在通过人工智能技术提供全天候的情感陪伴、专业的心理健康知识问答以及主动的情绪关怀，帮助大学生更好地管理情绪。

第二个备选项目为“智学——智能学习规划助手”。许多大学生在脱离了高中的高压环境后，难以自主坚持学习计划，且容易遗忘已学知识。该应用计划通过人工智能为用户制定个性化的学习与复习计划，并结合艾宾浩斯遗忘曲线进行定期的复习提醒，从而提升学习效率。

第三个备选项目是“食安——智能健康饮食助手”。当代大学生的饮食习惯往往不够规律，且缺乏专业的营养搭配知识。该项目构想通过多模态图像识别技术，让用户只需对食物拍照即可获取营养分析报告，并由人工智能提供科学的饮食建议，以改善大学生的身体健康状况。

1.3 最终选择
经过全面的评分与综合考量，我最终选择“心晴——大学生心理健康陪伴助手”作为本课程的实践项目。选择该项目的主要原因在于其具备极高的社会价值与现实需求，能够切实帮助大学生缓解心理压力。同时，该项目完美契合智能应用设计的七大核心原则，特别是能够深入应用“长期记忆”与“主动服务”这两个关键特性。在技术层面，目前开源社区拥有丰富的自然语言处理与情感分析模型资源，这为项目在规定周期内的顺利落地提供了坚实的技术保障。

第二部分：需求分析
2.1 项目概述
项目名称为“心晴——大学生心理健康陪伴助手”。本项目的目标用户是面临学业、就业、人际关系等压力的在校大学生群体。

用户面临的核心问题在于，大学生在遇到情绪低落或心理压力时，往往缺乏及时、隐秘且专业的倾诉渠道，导致负面情绪不断积累，进而引发更严重的心理健康问题。

本项目的解决方案是开发一款基于人工智能的陪伴助手。该系统通过自然语言对话理解用户情绪，提供温暖的情感支持与专业的心理疏导。系统将利用长期记忆功能记录用户的情绪波动历史与压力源，并通过主动服务机制在关键节点推送关怀信息与情绪调节建议。

2.2 用户故事
作为一名面临期末考试压力的大学生，我想要一个能够随时倾听我焦虑情绪的智能助手，以便于我能在深夜复习感到崩溃时获得即时的情感安慰与心理支持。

作为一名性格内向的大学生，我想要一个能够记住我过往烦恼与性格特点的数字朋友，以便于我在每次交流时无需重复解释自己的背景，能够直接进入深度的沟通与疏导。

作为一名经常忽视自身情绪状态的大学生，我想要系统能够根据我的对话频率与情绪倾向主动向我发送关怀提醒，以便于我能及时察觉自身的心理危机并采取有效的放松措施。

第三部分：概要设计
3.1 技术架构
本项目的技术架构分为表现层、业务逻辑层、智能服务层与数据存储层四个主要部分。表现层采用跨平台的Web前端技术，为用户提供流畅的对话界面与情绪可视化图表。业务逻辑层负责处理请求路由、会话管理以及定时任务调度。智能服务层是系统的核心，集成大语言模型、情感分析引擎以及记忆检索增强模块，实现对用户意图的精准理解与高质量的回复生成。数据存储层则结合关系型数据库与向量数据库，分别存储用户基础信息与对话向量，以支撑长期记忆功能。

3.2 技术选型
技术模块	选型方案	选型理由
前端框架	React + TypeScript	组件化开发效率高，生态丰富，能够快速构建响应式对话界面
后端框架	FastAPI (Python)	异步支持良好，性能优异，且与Python生态中的AI算法库无缝集成
大语言模型	OpenAI GPT-4 API	具备强大的自然语言理解与共情对话能力，适合处理复杂的情感交互
AI应用框架	LangChain	提供完善的Memory模块与链式调用封装，大幅降低大模型应用开发难度
向量数据库	Chroma	轻量级且易于部署，能够高效实现对话历史的语义检索与长期记忆提取
定时调度	APScheduler	灵活可靠的Python定时任务框架，满足主动关怀消息的定时推送需求

3.4 开源资源
本项目将充分复用现有的优秀开源资源以加速开发进程。第一个关键开源项目是LangChain，该开源框架提供了强大的组件来构建大模型应用，特别是其内置的各类记忆管理机制将直接用于实现本项目的长期记忆功能。第二个关键开源项目是Chroma，这是一个专为人工智能应用设计的开源嵌入式向量数据库，它将被集成到项目中用于高效存储和检索用户的历史对话向量，从而支撑系统对用户上下文的深入理解。

4.2 第一周任务
在项目的第一周，主要目标是完成前期的准备工作与基础设施的搭建。具体任务包括深入调研心理健康类应用的竞品以细化本项目的功能需求与交互流程，在本地计算机上安装并配置基础开发工具链，初始化前端与后端项目并建立版本控制仓库，以及获取大语言模型接口密钥并编写测试脚本以验证连通性。本周任务的验收标准为能够成功运行前后端的基础服务程序，模型接口测试脚本能够正常返回对话结果，且项目代码已成功推送到版本控制仓库中。
