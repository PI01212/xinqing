## 一、意图分类体系设计

### 1.1 大类划分

根据大学生心理健康陪伴场景，将用户意图划分为 **5个能力域**：

| 能力域 | 代码 | 核心能力描述 |
|--------|------|-------------|
| **情绪陪伴** | emotion | 记录情绪状态、分析情绪变化、提供情感支持 |
| **心理咨询** | counsel | 寻求心理建议、获取心理健康知识 |
| **压力管理** | stress | 记录压力源、获取减压方法、制定应对策略 |
| **个人成长** | growth | 目标规划、自我提升、习惯养成 |
| **智能对话** | chat | 日常闲聊、问候告别、帮助咨询 |

### 1.2 子意图定义

以下是AI识别意图的结果类别。不在类别中的消息当"普通对话"处理。

```
情绪陪伴（emotion）
   ├── emotion_log：记录当前情绪状态
   ├── emotion_query：查询历史情绪记录
   └── emotion_comfort：需要情感安慰与疏导

心理咨询（counsel）
   ├── counsel_ask：寻求具体心理问题的建议
   ├── counsel_knowledge：询问心理学相关知识
   └── counsel_technique：学习心理调节技巧

压力管理（stress）
   ├── stress_log：记录压力来源和程度
   ├── stress_relief：请求减压方法和放松技巧
   └── stress_plan：制定压力应对计划

个人成长（growth）
   ├── growth_goal：设定个人目标
   ├── growth_reflect：进行自我反思
   └── growth_habit：建立良好习惯

智能对话（chat）
   ├── chat_greeting：问候打招呼
   ├── chat_chat：日常闲聊
   ├── chat_help：询问助手功能
   └── chat_bye：告别离开
```

### 2.2 匹配规则说明

#### 匹配方式：精确匹配（使用 `^...$`）

**为什么选择精确匹配而不是包含匹配？**

| 对比项 | 精确匹配 `^你好$` | 包含匹配 `包含"你好"` |
|--------|------------------|---------------------|
| 输入："你好" | ✅ 正确匹配 | ✅ 匹配 |
| 输入："你好啊今天心情不好" | ❌ 不匹配（正确，交给AI处理） | ⚠️ 错误匹配（可能误拦截复杂消息） |
| 输入："跟你打个招呼你好" | ❌ 不匹配（正确） | ⚠️ 可能误匹配 |

**选择理由**：
1. **避免误拦截**：心理健康场景中，用户可能在长句中夹杂问候语，如"你好我今天考试没考好很难过"，如果用包含匹配会错误地只返回问候语，丢失用户的真实需求
2. **保证完整性**：精确匹配确保只有纯粹的简单寒暄才被第一层处理，有实际内容的消息都交给AI深入理解
3. **符合设计原则**：宁可漏掉也不要错判——漏掉的交给第二层AI处理，成本可控；错判的会导致用户体验下降

#### 长度限制：15个字

**为什么设置15字限制？**

| 消息长度 | 示例 | 是否适合第一层 | 原因 |
|----------|------|---------------|------|
| 2字 | "你好" | ✅ 适合 | 纯粹问候，无其他信息 |
| 5字 | "早上好呀" | ✅ 适合 | 时间问候，意图明确 |
| 10字 | "谢谢你的安慰" | ✅ 适合 | 表达感谢，无深层需求 |
| 15字 | "你在吗我想聊聊" | ⚠️ 边界 | 可能包含倾诉需求 |
| 20字 | "你好我今天真的很不开心" | ❌ 不适合 | 包含情绪表达，需AI理解 |
| 30字 | "最近考研压力很大晚上睡不着" | ❌ 不适合 | 复杂情绪表达，必须AI处理 |

**选择理由**：
1. **统计规律**：根据语言习惯，15字以内的消息通常是简单寒暄或短句确认
2. **场景适配**：心理健康陪伴场景下，用户的真实倾诉通常超过15字（如"最近期末考试压力特别大，感觉什么都复习不完"）
3. **成本平衡**：15字是一个合理的阈值，既能过滤大部分简单消息，又不会遗漏需要深度理解的短消息

```javascript
function fastMatch(message) {

  // 长度检查：超过15字通常不是简单对话
  if (text.length > 15) {
    return null;

  // 遍历所有规则进行匹配

    if (rule.patterns.some(pattern => pattern.test(text))) {
      return {
        matched: true,
        type: rule.type,
        response: rule.response,
        layer: 1  // 标记为第一层处理
      };
    }
  }

  return null;  // 未匹配，交给第二层AI处理
}
```

### 3.3 意图优先级规则

当一条消息可能匹配多个意图时，按以下优先级判断：

#### 优先级排序（从高到低）

```
优先级1：【危机预警】→ 如果检测到自伤、自杀、极端情绪等危险信号
         → 无论用户说什么，都优先标记为危机干预

优先级2：【情绪表达】→ emotion_comfort / emotion_log
         → 用户明确表达情绪词汇（难过、焦虑、抑郁、开心等）
         → 心理健康助手的核心价值就是情绪陪伴，应优先响应

优先级3：【具体诉求】→ counsel_ask / stress_relief / counsel_technique
         → 用户明确提出问题或请求（怎么办、怎么学、教我）
         → 这是有明确目标的求助，应该给予针对性回应

优先级4：【知识查询】→ counsel_knowledge / emotion_query / stress_identify
         → 用户询问信息类问题（什么是、为什么、怎么样）
         → 信息获取需求，相对次要但仍重要

优先级5：【记录类】→ emotion_track / stress_log / growth_goal
         → 用户要进行数据记录
         → 重要但可稍后处理，不影响即时体验

优先级6：【闲聊类】→ chat_chat / growth_reflect
         → 开放性对话，没有明确目的
         → 作为兜底，保持对话流畅性
```

| 用户消息 | 可能匹配的意图 | 最终判定 | 判定理由 |

```javascript
// ============================================================
// 心晴助手 - 第二层：AI意图识别模块
// ============================================================

const { INTENT_RECOGNITION_PROMPT } = require('./prompt-template');

/**
 * 调用大模型进行意图识别
 * @param {string} userMessage - 用户原始消息
 * @param {Object} userProfile - 用户画像
 * @returns {Promise<Object>} 意图识别结果
 */
async function recognizeIntent(userMessage, userProfile) {
  try {
    // 1. 构建提示词（填充用户画像变量）
    const prompt = buildPrompt(userMessage, userProfile);

    // 2. 调用大模型
    const response = await callLLM({
      model: 'gpt-4',  // 或 gpt-3.5-turbo（成本更低）
      messages: [{ role: 'user', content: prompt }],
      temperature: 0.3,  // 低温度保证稳定性
      max_tokens: 1000,
    });

    // 3. 解析JSON输出
    const result = parseJSONResponse(response.content);

    // 4. 后处理校验
    return postProcessResult(result);

  } catch (error) {
    console.error('意图识别失败:', error);
    // 解析失败时的降级处理
    return getFallbackResult(userMessage);
  }
}

/**
 * 构建完整提示词
 */
function buildPrompt(userMessage, profile) {
  return INTENT_RECOGNITION_PROMPT
    .replace('{userId}', profile.userId || '未知')
    .replace('{userProfile}', formatUserProfile(profile))
    .replace('{ageGroup}', profile.ageGroup || '18-25岁')
    .replace('{grade}', profile.grade || '未知')
    .replace('{major}', profile.major || '未填写')
    .replace('{lastEmotion}', profile.lastEmotion || '暂无记录')
    .replace('{lastEmotionTime}', profile.lastEmotionTime || '从未')
    .replace('{emotionTrend}', profile.emotionTrend || '数据不足')
    .replace('{commonStressors}', profile.commonStressors?.join('、') || '暂无')
    .replace('{interactionSummary}', profile.interactionSummary || '首次交互')
    .replace('{userMessage}', userMessage);
}

/**
 * 解析大模型的JSON响应（带容错处理）
 */
function parseJSONResponse(content) {
  try {
    // 尝试直接解析
    return JSON.parse(content);
  } catch (e) {
    // 尝试提取JSON部分（处理模型输出的多余文字）
    const jsonMatch = content.match(/\{[\s\S]*\}/);
    if (jsonMatch) {
      return JSON.parse(jsonMatch[0]);
    }
    throw new Error('无法解析JSON响应');
  }
}

/**
 * 后处理校验
 */
function postProcessResult(result) {
  // 校验必要字段
  if (!result.intent || !result.subIntent) {
    throw new Error('缺少必要字段');
  }

  // 置信度低于阈值时，降低为通用意图
  if (result.confidence < 0.7) {
    console.warn(`[低置信度] ${result.confidence}, 降级为通用意图`);
    result.subIntent = 'chat';
    result.reasoning += '（因置信度较低，降级为普通对话处理）';
  }

  // 危机级别标准化
  if (!['none', 'low', 'medium', 'high'].includes(result.crisisLevel)) {
    result.crisisLevel = 'none';
  }

  return result;
}

/**
 * 降级结果（当AI调用失败时）
 */
function getFallbackResult(message) {
  return {
    intent: 'chat',
    subIntent: 'chat',
    confidence: 0.3,
    reasoning: '意图识别服务暂时不可用，降级为普通对话处理',
    crisisLevel: 'none',
    extractedData: {},
    safetyCheck: {
      needReview: false,
      warning: null,
      suggestedAction: 'normal'
    }
  };
}

module.exports = { recognizeIntent };
```

## 四、第三层：安全检查规则

### 4.1 为什么需要第三层？

> **第二层 AI 识别 ≠ 第三层安全检查**
>
> 在心理健康领域，安全性至关重要：
> - **AI 可能漏检**：模型幻觉、对抗性攻击可能导致危机信号被忽略
> - **AI 可能误判**：不同模型、不同温度下的判断不一致
> - **法律责任**：心理健康涉及人身安全，必须有确定性的保护机制
>
> **结论**：AI 输出的安全检查结果仅作参考，**代码必须独立执行强制性的安全规则**。

```javascript
function safetyCheck(intentResult, userMessage, userProfile) {
  const { absoluteForbidden, conditionalWarning, suggestProfessionalHelp, crisisResources } = SAFETY_RULES;

  // ════════════════════════════════════════════════════════

  if (warnResult.warning) {

    return {

      level: 'WARN',
      warningResponse: warnResult.response,
      reason: warnResult.reason,
      warningCategory: warnResult.category

    return {
      pass: true,
      level: 'URGENT',
      suggestionResponse: generateProfessionalHelpResponse(),
      reason: urgentResult.reason
    };
  }

  // ════════════════════════════════════════════════════════

    if (intentResult.safetyCheck.warning.includes('【必须拦截】')) {

        type: 'harmOthers',
        reason: `检测到伤害他人意向关键词：「${keyword}」`,

}

        category: 'anxiety',
        keyword: keyword,
        response: rules.anxiety.responseTemplate(keyword),
        reason: `检测到可能的焦虑症状关键词：「${keyword}」`

        response: rules.acuteStress.responseTemplate(),
```

### 4.7 完整调用流程

```javascript
// ============================================================
// 心晴助手 - 三级架构完整调用流程
// ============================================================

const { fastMatch } = require('./layer1-fast-match');
const { recognizeIntent } = require('./layer2-ai-recognition');
const { safetyCheck } = require('./layer3-safety-check');

/**
 * 处理用户消息的主入口函数
 *
 * @param {string} userMessage - 用户输入的消息
 * @param {Object} userProfile - 用户画像
 * @returns {Promise<string>} 最终回复给用户的内容
 */
async function handleMessage(userMessage, userProfile) {

  // ════════════════════════════════════════════════════════
  // 第一层：快速匹配（正则，零成本，<5ms）
  // ════════════════════════════════════════════════════════
  const fastResult = fastMatch(userMessage);
  if (fastResult && fastResult.matched) {
    console.log(`[第一层命中] 类型: ${fastResult.type}`);
    return fastResult.response;  // 直接返回，不调用AI
  }

  // ════════════════════════════════════════════════════════
  // 第二层：AI意图识别（500ms-1s）
  // ════════════════════════════════════════════════════════
  let intentResult;
  try {
    intentResult = await recognizeIntent(userMessage, userProfile);
    console.log(`[第二层识别] 意图: ${intentResult.intent}/${intentResult.subIntent}, 置信度: ${intentResult.confidence}`);
  } catch (error) {
    console.error('[第二层失败]', error.message);
    // AI调用失败时的降级处理
    return "抱歉，我暂时无法理解你的意思。能换个说法再说一次吗？或者你可以试试简单的问候，我会尽力帮助你。";
  }

  // ════════════════════════════════════════════════════════
  // 第三层：安全检查（代码强制执行，<1ms）
  // ════════════════════════════════════════════════════════
  const safetyResult = safetyCheck(intentResult, userMessage, userProfile);
  console.log(`[第三层安全] 级别: ${safetyResult.level}, 通过: ${safetyResult.pass}`);

  // ════════════════════════════════════════════════════════
  // 根据安全检查结果决定响应策略
  // ════════════════════════════════════════════════════════

  // 🔴 BLOCK：拦截，返回危机干预信息
  if (!safetyResult.pass && safetyResult.level === 'BLOCK') {
    return safetyResult.response;
  }

  // ⚠️ WARN / 🟠 URGENT：允许执行，但附加提示
  if (safetyResult.level === 'WARN' || safetyResult.level === 'URGENT') {
    // 执行正常的意图处理
    const actionResponse = await executeIntent(intentResult, userProfile);

    // 合并安全提示
    const extraTip = safetyResult.warningResponse || safetyResult.suggestionResponse;
    return `${actionResponse}\n\n───\n\n${extraTip}`;
  }

  // ✅ PASS：正常执行
  if (safetyResult.pass && safetyResult.level === 'PASS') {
    return await executeIntent(intentResult, userProfile);
  }
}

/**
 * 执行识别到的意图
 */
async function executeIntent(intentResult, userProfile) {
  const { intent, subIntent, extractedData } = intentResult;

  switch (`${intent}_${subIntent}`) {
    case 'emotion_log':
      return handleEmotionLog(extractedData, userProfile);
    case 'emotion_comfort':
      return handleEmotionComfort(extractedData, userProfile);
    case 'counsel_ask':
      return handleCounselAsk(extractedData, userProfile);
    case 'stress_relief':
      return handleStressRelief(extractedData, userProfile);
    // ... 其他意图处理
    default:
      return handleGeneralChat(intentResult, userProfile);
  }
}
```

## 五、学习反思与AI贡献度

### 5.1 学习收获

通过本次三级意图识别方案的设计与实践，我获得了以下收获：

#### 知识层面

1. **深刻理解了分层架构的设计思想**
   
   以前我认为所有用户输入都应该交给AI处理，现在我明白了为什么要分层：
   - 第一层用正则处理简单消息，**零成本、毫秒级响应**
   - 第二层用AI处理复杂语义，**灵活但有一定成本**
   - 第三层用代码做安全检查，**确定性保障**
   
   这种"能用简单方法就不用复杂方法"的思想，体现了优秀的工程思维。

2. **掌握了提示词工程的核心技巧**
   
   设计AI意图识别的提示词时，我学会了：
   - 用表格清晰列出所有可选意图，防止AI乱编
   - 给出完整的示例（few-shot learning），比纯文字描述有效得多
   - 设置低temperature（0.3）以保证输出稳定性
   - 设计结构化的JSON输出格式，便于程序解析
   - 加入置信度机制，不确定时降级处理

3. **认识到安全设计的重要性**
   
   特别是在心理健康这个敏感领域：
   - AI不能承担法律责任，所以必须有代码层面的强制性安全检查
   - 安全规则要分级：绝对禁止 > 条件警告 > 建议就医
   - 要考虑到最坏的情况（如用户表达自杀意念），并有完善的危机干预流程

#### 能力层面

1. **系统能力**：学会了从整体架构角度思考问题，不再是"写一个函数"，而是"设计一个系统"
2. **分析能力**：能够根据业务场景（大学生心理健康）抽象出合适的意图分类体系
3. **工程能力**：掌握了正则表达式、JSON结构设计、容错处理等实用技能
4. **文档能力**：学会了编写清晰、完整的技术文档，包括表格、代码示例、测试用例

### 5.2 困难与解决

#### 困难1：意图分类的粒度把握

**问题**：一开始我想分得很细，比如把"学业压力""就业压力""人际压力"分成不同的意图，结果发现：
- 意图太多，AI容易混淆
- 很多情况下用户的消息同时涉及多个方面

**解决**：参考课程案例，采用"大类+子意图"的两级结构：
- 大类控制在5-6个（如emotion、counsel、stress等）
- 子意图根据具体场景细化
- 具体细节通过extractedData提取，不占意图分类

#### 困难2：安全规则的边界划定

**问题**：在心理健康场景下，什么样的关键词应该拦截？什么样的只是警告？
- 太严格：可能误拦正常倾诉（如"烦得想死"可能是夸张表达）
- 太宽松：可能漏掉真正的危机信号

**解决**：
- 采用**三级分类**（BLOCK/WARN/URGENT）而不是简单的二元判断
- BLOCK只用于**明确的自伤/自杀/伤人**表述
- WARN用于**症状描述**（抑郁、焦虑等）
- URGENT用于**就医相关**表述
- 同时保留AI的safetyCheck作为补充，双重保险

#### 困难3：正则匹配的精度控制

**问题**：第一层的正则应该用精确匹配还是包含匹配？
- 精确匹配：可能漏掉一些合理的简单消息
- 包含匹配：可能错误拦截复杂消息中的问候语

**解决**：
- 选择**精确匹配**（^...$），宁可漏也不要错
- 设置**15字长度限制**，短消息才进入第一层
- 漏掉的交给第二层AI处理，成本可控

### 5.3 AI贡献度说明

本次作业中，AI在我的工作中的作用和贡献如下：

| 工作环节 | 我的贡献 | AI的贡献 |
|----------|---------|---------|
| **项目理解** | 100% - 基于我自己写的选题报告，我完全理解项目背景和需求 | 0% - AI不了解我的项目 |
| **意图分类设计** | 70% - 我根据心理健康场景设计了5大类26个子意图的分类体系 | 30% - AI帮助优化分类命名和层级结构 |
| **第一层词库设计** | 60% - 我根据大学生口语习惯设计了82个关键词 | 40% - AI帮助补充了一些口语变体和中英文化 |
| **第二层提示词设计** | 50% - 我设计了提示词的整体结构和内容框架 | 50% - AI帮助完善了提示词的表达、示例和细节 |
| **第三层安全规则** | 80% - 我基于心理健康专业知识设计了安全规则体系 | 20% - AI帮助整理格式和完善响应模板 |
| **代码实现** | 70% - 我编写了核心逻辑和架构 | 30% - AI帮助优化代码结构和添加注释 |
| **学习反思** | 100% - 这是我的真实感悟 | 0% - AI无法代替我的思考 |

**总体AI贡献度：约35%**

**具体说明**：
- AI主要扮演了**辅助工具**的角色，帮助我：
  - 优化文字表达的准确性和流畅性
  - 补充我没想到的边界情况
  - 格式化和规范化文档结构
  - 生成代码框架和示例
- **核心设计和决策都是我自己完成的**：
  - 意图分类体系是基于我对大学生心理健康需求的深入理解
  - 安全规则是结合心理健康专业知识设计的
  - 所有设计决策都有明确的理由和考量

### 5.4 方案改进方向

我觉得目前的方案还有以下可以改进的地方：

1. **第一层词库的动态扩展**
   - 当前词库是静态的82个关键词
   - 未来可以根据实际用户数据，用数据分析发现高频简单消息，动态扩充词库
   - 可以加入A/B测试，验证哪些关键词放第一层效果更好

2. **第二层提示词的持续优化**
   - 可以收集"AI识别错误"的case，加入到提示词的反例中
   - 可以针对不同年级、不同专业的学生微调提示词（如大一新生 vs 毕业班学生关注点不同）
   - 可以引入Few-shot learning，用真实的优质对话案例作为示例

3. **第三层安全规则的智能化**
   - 目前是硬编码的关键词匹配，可以考虑用轻量级ML模型辅助判断
   - 可以加入用户历史行为分析（如某用户近期多次出现抑郁关键词，自动提升关注等级）
   - 可以接入实时的危机干预专家系统（在BLOCK级别时）

4. **整体架构的性能优化**
   - 第一层可以用Trie树等数据结构优化匹配速度（虽然目前已经很快）
   - 第二层可以考虑缓存常见意图的识别结果
   - 可以加入意图识别结果的置信度反馈循环，不断优化模型

5. **跨平台适配**
   - 当前方案主要针对文本输入，未来需要适配语音输入（语音转文本后再进入流水线）
   - 可以考虑加入图像/表情符号的情绪识别（如用户发送😭表情）
