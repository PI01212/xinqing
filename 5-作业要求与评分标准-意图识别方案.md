# 作业要求与评分标准 — 意图识别方案

## 一、作业背景

在本次课程中，我们学习了意图识别的三层架构设计，并以"50+女性运动健康助手"为案例进行了完整演示：

- **第一层**：正则快速匹配，零成本处理无歧义的简单消息
- **第二层**：AI提示词识别，处理复杂语义并提取参数
- **第三层**：代码安全检查，强制规则兜底保护

现在，请结合你的个人项目，制定属于你的三级意图识别方案。

### 总体要求

| 要求项        | 说明                       |
| ---------- | ------------------------ |
| **结合个人项目** | 必须基于你自己的项目场景，不能照搬50+助手案例 |
| **三级完整**   | 必须包含一级、二级、三级三个层次的完整设计    |
| **具体可执行**  | 词库、模板、规则都要具体，不能只有概念描述    |
| **格式规范**   | 按照提供的模板格式填写，结构清晰         |

## 三、评分标准

### 3.1 总分构成

| 部分          | 分值       | 占比       |
| ----------- | -------- | -------- |
| 第一层：正则匹配词库  | 30分      | 30%      |
| 第二层：AI提示词模板 | 40分      | 40%      |
| 第三层：安全检查规则  | 20分      | 20%      |
| 整体规范与创新     | 5分       | 5%       |
| 学习反思        | 5分       | 5%       |
| **合计**      | **100分** | **100%** |

### 3.2 详细评分标准

#### 第一层：正则匹配词库（30分）

| 评分项     | 分值  | 评分标准                            |
| ------- | --- | ------------------------------- |
| 词库分类完整性 | 10分 | 分类合理，每类≥5个关键词得满分；分类不足或词数不足扣2-5分 |
| 匹配规则合理性 | 8分  | 规则说明清晰，有理有据得满分；说明不清扣2-4分        |
| 词库统计准确性 | 6分  | 统计准确，覆盖场景说明清楚得满分                |
| 格式规范性   | 6分  | 格式规范、表格清晰得满分；格式混乱扣2-3分          |

#### 第二层：AI提示词模板（40分）

| 评分项      | 分值  | 评分标准                    |
| -------- | --- | ----------------------- |
| 身份定义清晰度  | 6分  | 身份、服务对象、核心价值明确得满分       |
| 意图分类完整性  | 12分 | ≥3大类、每类≥2子意图得满分；不足扣3-6分 |
| 优先级规则合理性 | 6分  | 规则合理、说明清晰得满分            |
| 数据结构规范性  | 8分  | 结构清晰、字段说明完整得满分          |
| 示例完整性    | 8分  | ≥2个完整示例、JSON格式正确得满分     |

#### 第三层：安全检查规则（20分）

| 评分项      | 分值  | 评分标准                     |
| -------- | --- | ------------------------ |
| 禁止关键词完整性 | 8分  | ≥3类、每类≥3个关键词得满分          |
| 警告关键词合理性 | 6分  | 场景合理、处理方式明确得满分           |
| 风险类别清晰度  | 6分  | BLOCK/WARN/URGENT说明清晰得满分 |

#### 整体规范与创新（5分）

| 评分项    | 分值  | 评分标准                    |
| ------ | --- | ----------------------- |
| 结合个人项目 | 3分  | 紧密结合个人项目场景得满分；照搬案例扣2-3分 |
| 创新与亮点  | 2分  | 有创新设计或亮点得满分             |

#### 学习反思（5分）

| 评分项    | 分值  | 评分标准                 |
| ------ | --- | -------------------- |
| 学习收获反思 | 5分  | 认真思考学习收获和进步，有真实感悟得满分 |

**说明**：学习反思部分不区分对错，只要认真思考并写出自己的学习收获、知识掌握情况、能力提升等真实感悟即可得满分。目的是帮助同学们回顾学习过程，巩固所学知识。

### 3.3 评分等级

| 等级  | 分数范围    | 说明              |
| --- | ------- | --------------- |
| 优秀  | 90-100分 | 三级完整、设计合理、有创新亮点 |
| 良好  | 80-89分  | 三级完整、设计基本合理     |
| 中等  | 70-79分  | 三级基本完整、部分设计不够合理 |
| 及格  | 60-69分  | 三级不完整或设计有明显问题   |
| 不及格 | <60分    | 严重不完整或照搬案例      |

## 五、学习反思

### 5.1 反思要求

请在作业最后添加一个简短的学习反思，思考以下问题：

1. **知识收获**：通过本次作业，你学到了哪些新知识？
2. **能力提升**：你的哪些能力得到了提升？
3. **困难与解决**：完成作业过程中遇到了什么困难？如何解决的？
4. **改进方向**：你觉得自己的方案还有哪些可以改进的地方？

### 5.2 说明

- 本部分**不区分对错**，只要认真思考并写出真实感悟即可
- 目的是帮助同学们回顾学习过程，巩固所学知识
- 请诚实反思，不要为了得分而编造内容

### 5.3 示例

```
## 学习反思

通过本次作业，我深入理解了意图识别的三层架构设计。之前只知道要用AI来理解用户意图，
现在明白了为什么要分层：第一层处理简单消息可以节省成本，第二层处理复杂语义，
第三层保障安全。

最大的收获是学会了如何设计安全检查规则。我的项目是儿童学习助手，需要特别注意
保护儿童隐私和过滤不当内容。通过设计第三层规则，我意识到安全设计的重要性。

遇到的困难是如何确定意图分类的粒度。一开始分得太细，后来参考课程案例，
学会了"大类+子意图"的方式，问题就解决了。

我觉得我的方案还可以改进：第一层词库可以根据实际用户数据持续扩充，
第三层安全规则也需要根据实际case不断更新。
```
